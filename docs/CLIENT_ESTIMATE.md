# LinkedIn Marketplace & Rental — пропозиція

**Що ми робимо**: онлайн-сервіс, де клієнти купують SMM-послуги (наприклад, набір connections у LinkedIn) і орендують готові LinkedIn-акаунти. Адмін керує всім з однієї панелі.

**Термін**: ~4 тижні до запуску в роботу.

---

## Що отримає клієнт у MVP

### Тиждень 1 — Сайт + основа

| Що | Як це бачить юзер |
|---|---|
| **Лендінг (EN + UA)** | Головна з двома кнопками: "Купити SMM-послуги" / "Орендувати акаунт". Блок "Як це працює", FAQ, контакти. Тексти редагуються в адмінці. |
| **Реєстрація і логін** | Юзер реєструється через email, отримує лист підтвердження, може скинути пароль. |
| **Захист даних акаунтів** | Усі паролі, проксі і 2FA-секрети зберігаються зашифрованими (AES-256). Невидимо для юзера — захищає бізнес. |
| **Інфраструктура** | Налаштовано базу даних, домен, деплой, окремий сервер для авто-задач. |

### Тиждень 2 — Маркетплейс

| Що | Як це бачить юзер |
|---|---|
| **Каталог SMM-послуг** | Сторінка `/services` з картками послуг. Старт — Connections, далі легко додавати Followers / Likes / Comments. Пакети 25 / 50 / 75 / 100. Якщо немає вільних акаунтів — великий пакет неактивний. |
| **Каталог акаунтів для оренди** | Сторінка `/rent` з картками акаунтів: гео, кількість connections, ціна. Фільтри. Можна обрати поштучно або готовим пакетом. |
| **Форма замовлення** | Юзер залишає заявку → одразу email + Telegram адміну з кнопками "Прийняти / Відхилити". |
| **База акаунтів в адмінці** | Адмін додає акаунти вручну або через CSV-імпорт. Поля: login, password, proxy, 2FA, гео, статус, пул (SMM / Rental). |

### Тиждень 3 — Кабінет + сповіщення

| Що | Як це бачить юзер |
|---|---|
| **Особистий кабінет** | "Мої замовлення" — історія. "Мої акаунти" — кнопки: показати login/proxy, показати 2FA-код (генерується на льоту), продовжити оренду, поскаржитися. |
| **Адмін-дашборд** | Скільки акаунтів у кожному пулі, скільки готових, у роботі, забанено. Bulk-дії, перегляд "хто і до коли орендує", ручна заміна. |
| **Email-сповіщення (8 типів)** | Підтвердження заявки, видача акаунтів, нагадування за 3 і 1 день до експірації, експірація, авто-заміна забаненого. |
| **Telegram-бот для адміна** | Сповіщення про нові заявки, бан акаунтів, низький запас. Кнопки "Прийняти / Відхилити" прямо в Telegram. |

### Тиждень 4 — Автоматика + запуск

| Що | Як це бачить юзер |
|---|---|
| **Авто-зміна пароля при експірації** | Сервіс заходить на LinkedIn, змінює пароль, оновлює 2FA. Старі дані юзера стають невалідними. |
| **Інтеграція з ADSPower / LinkedinHelper** | Сервіс раз на годину перевіряє стан акаунтів у ADSPower: коли прогрітий — автоматично переключає статус на "готовий". |
| **Авто-заміна забаненого** | Якщо акаунт забанили під час оренди — автоматично видається інший з того ж пулу і гео. |
| **Тестування і доведення** | Перевірка ключових сценаріїв, мобільної версії, виправлення дрібних багів. |
| **Production-запуск** | Деплой сайту + воркера, налаштування БД, інструкція для адміна. |

## Що може зсунути термін

| Фактор | Вплив |
|---|---:|
| LinkedIn змінив UI — Playwright-скрипти треба переписувати | +0.5-1 тиждень |
| Клієнт хоче кастомний дизайн з кількома раундами правок | +1 тиждень |
| Нестабільні проксі | +0.5 тижні |
| ADSPower вже працює і налаштований у клієнта | **−0.5 тижні** |

---

## Що буде гарантовано на запуск

- Працюючий лендінг (EN + UA)
- Реєстрація юзерів + кабінет
- Каталог SMM-послуг з capacity-перевіркою
- Каталог акаунтів з фільтрами
- Email + Telegram сповіщення з inline-кнопками для адміна
- Reveal credentials + кнопка "Show 2FA code" у кабінеті
- Auto-нагадування 3д / 1д + auto-зміна паролю при експірації
- Auto-заміна забаненого акаунта
- ADSPower sync
- Production deploy + інструкція для адміна

## Дорожня карта після MVP

| Фаза | Що додаємо | Орієнтовно |
|---|---|---:|
| **MVP** (зараз) | Все вище | ~4 тижні |
| **Автоматика SMM-дій** | Бот виконує connections / likes / comments з anti-bot логікою (5 entry paths, warm-up, failure injection 5-10%) | +6-10 тижнів |
| **Платежі** | Stripe / NOWPayments, авто-фактури | +2-3 тижні |
| **Мобільний застосунок** (PWA) | Зручний доступ клієнтів з телефону | +3-4 тижні |
| **Multi-tenant** | Reseller-модель | +4-6 тижнів |
